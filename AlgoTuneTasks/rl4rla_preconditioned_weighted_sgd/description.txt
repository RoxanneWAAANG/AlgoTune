Preconditioned Weighted SGD (Sketch-and-Precondition) Task

Based on: RL4RLA (ICML submission) — curriculum "leverage_score_subsampling",
config DiscoverNLA/hp/mcgs_ucd/leverage_score_subsampling/6.yml.

Given an overdetermined least-squares problem A x ≈ b where A ∈ R^{m×n} (m >> n)
has a MID_COND spectrum (chi-squared distributed singular values) and high-leverage
row structure (multivariate-t left singular vectors), find x* that minimises
(1/2) ||A x - b||².

The RL agent discovers "leverage score subsampling" — a sketch-and-precondition
stochastic gradient descent algorithm — starting from plain gradient descent.
The target algorithm:
  1. [SETUP] Compute a sketch-based preconditioner:
       SA   = S A   where S is a Sparse Johnson-Lindenstrauss Transform (SJLT),
                    sketch dimension d = sampling_factor × n
       R    = R-factor from QR(SA)   (n × n upper triangular)
       P    = (R^T R)^{-1}           (preconditioner ≈ (A^T A)^{-1})
  2. [SETUP] Compute statistical leverage scores: lev_i = ||A_i||² / ||A||_F²
  3. [EACH ITERATION] Sample batch_size rows S_t ~ lev (leverage-score sampling):
       g_t  = A_{S_t}^T (A_{S_t} x_t - b_{S_t})    (stochastic gradient)
       x_{t+1} = x_t - lr × P g_t                   (preconditioned update)

This is the algorithm discovered by RL4RLA via Monte Carlo Graph Search (MCGS),
improving upon plain gradient descent by orders of magnitude on ill-conditioned
systems.

System parameters (n = number of columns, problem size):
  num_rows         = n × 200    (= 10 000 when n = 50)
  spectrum         = chi-squared (df=1) singular values
  left sing. vecs  = multivariate-t QR (df=0.1), high statistical leverage
  right sing. vecs = Gaussian orthonormal QR
  prop_range       = 1.0  (b lies entirely in column space of A → consistent system)
  sampling_factor  = 8,   batch_size = 100
  max_iterations   = 10,  learning_rate = 0.5

Input: A dictionary with keys:
  - "A":              numpy array of shape (num_rows, num_cols), the feature matrix
  - "b":              numpy array of shape (num_rows,), the right-hand side
  - "x_opt":          numpy array of shape (num_cols,), the pseudoinverse solution
  - "num_rows":       int, number of rows m
  - "num_cols":       int, number of columns n (= problem size parameter)
  - "sampling_factor": int (8), sketch dimension = sampling_factor × num_cols
  - "batch_size":     int (100), rows sampled per iteration
  - "max_iterations": int (10), number of solver iterations
  - "learning_rate":  float (0.5), step size

Output: A dictionary with key:
  - "x": numpy array of shape (num_cols,), the approximate solution

Example input (n=3, illustrative):
{
    "A": [[2.1, 0.3], [0.1, 1.8], [1.5, 0.9], ...],
    "b": [1.2, 0.8, 1.1, ...],
    "num_rows": 600, "num_cols": 3,
    "sampling_factor": 8, "batch_size": 100,
    "max_iterations": 10, "learning_rate": 0.5
}

Example output:
{
    "x": [0.42, 0.31, 0.18]
}

Verification: ||A x - b|| / ||b|| < 0.01

Category: matrix_operations
